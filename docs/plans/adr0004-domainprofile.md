> **临时文档**:这是 ADR 0004（结构化通道全通用化）的逐步实现脚手架，重构完成后删除。耐久决策见 `docs/adr/0004-domain-profile-generalization.md`；实现以代码 + 测试 + 每步一个 commit 为准。无 `covers:` → 漂移豁免。

## 存储选型
Keep the fixed-typed-column table named `fact_metric` exactly as today (all 10 base + 8 v2/temporal columns retained, in declaration order) and ADD ONE nullable `dim_key TEXT` column carrying a canonical sorted-JSON natural key over IDENTITY dims only (metric,entity,channel,period=f'{period_type}{period}'; geography is identity=False, stays a non-key overwritable column). The UNIQUE constraint moves from the 5-col composite to UNIQUE(dim_key) (single TEXT equality, still 0-or-1 rows -> preserves anti-fabrication determinism and idempotent upsert), with the old composite ux_fact_metric index kept alongside for the green window. Arbitrary-dimension capability is achieved by ADDING (not replacing) an in-memory Fact.dimensions bag (field 19, default_factory=dict, reserved-name-guarded so it can never shadow a structural/lineage column or `tags`), excluded from DB columns for the finance window since all finance dims are already typed columns. REJECTED: EAV (partial-match could return >1 row -> ambiguous number reaches the model, breaks determinism; can't UNIQUE an N-row tuple); pure JSON blob (SELECT */PRAGMA readers in test_t2/test_temporal/ingestion_manifest read typed columns by name); rename fact_metric->fact+VIEW (test_t2 CREATEs a literal TABLE fact_metric, 10+ files read the literal name); query(dimensions:dict) public surface (all ~20 callers positional — immediate red wall). Justification: dim_key reproduces the old (metric,entity,period_type,period,channel) uniqueness exactly; lineage stays FIXED NOT NULL columns never in the blob/key; period_type-prefixed serialization is injective across FY/HY/Q grains; migration is the proven idempotent ALTER-add-column + per-row Python backfill path (no rename/VIEW/DROP), so test_t2 legacy DB migrates non-destructively.

## 非金融第二域（证明通用性）
lab_metrology (materials/QA testing lab) — a deliberately NON-financial, NO-temporal domain. Dimensions: measurement (measure analogue: units MPA/HRC/PCT, clarify='ask_first', whitelist_in_fabrication_check=False so a measure value is NEVER whitelisted), site (entity/scope-anchor analogue, matched on SecurityGate masked_text), batch (categorical identity dim with NO temporal axis — proves period is not privileged), specimen (channel analogue: optional, default='TOTAL', non-expanding), region (derived_from='site', SKIPPED from tool properties exactly like geography). external_entities lists rival labs so the deterministic SecurityGate refuses competitor queries unchanged. Example: '上海实验室 批次A 的抗拉强度是多少' -> dims {measurement:TENSILE, site:SH_LAB, batch:BATCH_A, specimen:TOTAL, region:<derived>}; found -> value+source(provenance); not_found -> deterministic 查不到 (anti-fabrication); a rival-lab query -> SecurityGate refusal — every hard invariant holds with arbitrary non-temporal dimensions. Because it has no kind='temporal' dim, detect_fabricated_numbers strips NOTHING for the lab profile (strictly safer: every digit in a refusal answer is flagged), which is the strongest possible proof the period-whitelist was never widened. Shipped as a constructed DomainProfile test fixture + separate lab golden/baseline; the finance qa_baseline.json (n_cases=41) is byte-untouched.

### lens: invariants — verdict=needs-revision
- **问题**: ANTI-FABRICATION WIRING GAP (Step 11 + Step 13). The plan makes detect_fabricated_numbers' period-strip regex profile-sourced and claims the lab domain (no temporal dim) 'strips NOTHING (safer)'. - **问题**: PERIOD-WHITELIST WIDENING RISK (Step 11).
